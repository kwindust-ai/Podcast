## 📖 Episode Detail - Main Features

### Two Main Tabs

#### Tab 1: TIMESTAMPED UPDATES ⏰
```
Purpose: Post updates tied to specific moments in episode

How to post an update:
1. Find the form: "Post Timestamped Update"
2. Enter timestamp: Where in episode was this discussed?
   Example: "12:45" (12 minutes, 45 seconds)
3. Enter content: What recent development happened?
   Example: "DNA lab results confirm match with 99.9% accuracy"
4. Click: "Post Update at 12:45"
5. Done! Update appears in feed below

When listeners replay episode at 12:45, they see the update
```

**Example Timeline:**
```
Episode: "Blackwell Case: Part 1" (58 minutes long)

00:00 ➜ Introduction
05:00 ➜ Initial case details
12:45 ➜ (Episode discusses DNA evidence)
       ⚡ UPDATE APPEARS: "DNA Evidence Confirmed"
       "Lab results show DNA match with 99.9% accuracy"
28:30 ➜ (Episode discusses witness sightings)
       📢 UPDATE APPEARS: "Witness Statement Update"
       "New witness came forward claiming to have seen suspect"
40:00 ➜ Expert analysis
58:00 ➜ Outro/Preview of next episode
```

**Update Types & Colors:**
```
⚡ BREAKING (Red)
   Use for: Major arrests, verdicts, DNA matches
   Example: "Suspect arrested at home after DNA match"

🚨 MAJOR (Orange)
   Use for: Significant events, confessions
   Example: "Suspect confesses but claims self-defense"

📢 UPDATE (Blue)
   Use for: General case developments, witness statements
   Example: "New witness comes forward with timeline"

✍️ CREATOR (Purple)
   Use for: Corrections, clarifications
   Example: "Correction: DNA results arrived in 5 weeks not 6"
```

## 💡 Pro Tips

### For Case Coverage:
1. **Post updates immediately** when news breaks
2. **Reference the timestamp** where you discussed it
3. **Keep it factual** - source your updates
4. **Monitor comments** - they show what audiences care about
5. **Use colors strategically** - breaking news (⚡), major events (🚨)

### For Growing Audience:
1. **Update frequently** during active trials
2. **Respond to top comments** (high likes show interest)
3. **Post multiple updates** per episode if developments happen
4. **Check completion rates** - high rates mean listeners stay engaged
5. **Track new subscribers** - see which episodes convert best

### For Quality:
1. **Accurate timestamps** - listeners will use them
2. **Clear content** - don't assume listeners remember exact details
3. **Source documentation** - link to court documents when possible
4. **Timeline updates** - show how case evolved over time

## 🔧 Troubleshooting

**"I can't find an episode"**
→ Use search feature instead of scrolling
→ Try different keywords

**"Updates aren't appearing"**
→ Scroll down in the Updates tab
→ Updates appear in reverse chronological order (newest first)

**"I need to edit an update I posted"**
→ Current version doesn't support editing
→ Can post a correction as new update

**"The timestamp format is wrong"**
→ Use MM:SS format (e.g., "12:45" for 12 minutes 45 seconds)
→ Not HH:MM:SS - just minutes and seconds

**"Comments aren't loading"**
→ Hard refresh the page (Cmd+Shift+R on Mac)
→ Check internet connection
